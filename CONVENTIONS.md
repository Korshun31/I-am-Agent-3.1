# Правила и порядок в проекте I am Agent

## Общие принципы

1. **Чистота кода** — не допускать разрастания «помойки»: лишние файлы, неиспользуемые зависимости, копипаста.
2. **Порядок в структуре** — новые экраны в `src/screens/`, новые компоненты в `src/components/`, сервисы в `src/services/`.
3. **Документация** — обновлять `README.md`, `CONTEXT_FOR_AI.md` и `CONVENTIONS.md` при существенных изменениях.

## Структура

- `src/screens/` — только экраны (страницы навигации).
- `src/components/` — переиспользуемые компоненты и модалки.
- `src/services/` — работа с хранилищем, API, утилиты.
- `src/context/` — React Context (язык, тема и т.п.).
- `src/i18n/` — переводы.
- `assets/` — иконки, изображения.

## Именование

- Файлы компонентов: PascalCase (`AccountScreen.js`, `LanguageModal.js`).
- Сервисы: camelCase (`agentsStorage.js`).
- Ключи переводов: camelCase (`myDetails`, `callOrMessage`).

## Добавление нового функционала

1. Новый экран → `src/screens/`.
2. Новая модалка/компонент → `src/components/`.
3. Новый сервис или работа с данными → `src/services/`.
4. Новые строки интерфейса → `src/i18n/translations.js` (en, th, ru).
5. После добавления — обновить `CONTEXT_FOR_AI.md` при необходимости.

## Отступы в блоках

**Верхний отступ** — от верхнего края блока до верхней точки первой строки (верхний пиксель иконки в заголовке).  
**Нижний отступ** — от нижнего края блока до нижней точки последней строки.

**Правило:** верхний и нижний отступы блока всегда равны (`BLOCK_VERTICAL_PADDING`).  
**Внутренние отступы** между строками (header↔контент, контент↔контент, последняя строка↔кнопка) тоже равны между собой (`BLOCK_ROW_GAP`).

Константы заданы в `AccountScreen.js`. При добавлении новых блоков — использовать их.

## Что не делать

- Не добавлять неиспользуемые файлы и зависимости.
- Не хранить пароли в UI — только в `agentsStorage`, в объекте агента.
- Не дублировать логику: выносить в сервисы или хуки.
