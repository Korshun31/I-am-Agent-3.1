# Сквозные правила: Error Handling (Ошибки)

> Эти правила действуют во ВСЕХ модулях проекта.

## EH-1. Ошибки на экране, не во всплывающем окне

Ошибки показываются **на экране** (inline) — красный текст под кнопкой или под полем. Не во всплывающем окне (Alert).

Пользователь видит ошибку и сразу видит где исправить.

**Исключение:** подтверждение удаления — это Alert/confirm, потому что это не ошибка а вопрос "ты уверен?".

## EH-2. Текст на языке пользователя

Текст ошибки — **на языке пользователя** через `t(key)`. Не на английском, не технический текст из базы данных.

Пользователь не должен видеть `BOOKING_CONFLICT` — он должен видеть "Эти даты уже заняты".

## EH-3. Коды ошибок от сервера

Ошибки от Supabase перехватываются в сервисе и преобразуются в понятный код (`BOOKING_CONFLICT`, `EMAIL_EXISTS`, `BOOKING_NO_COMPANY`). Экран получает код и показывает
локализованное сообщение через `t()`.
