# Сквозные правила: i18n (Переводы)

> Эти правила действуют во ВСЕХ модулях проекта.

## I18N-R1. Все три языка одновременно

Каждая новая строка текста в интерфейсе добавляется во **все три языка** одновременно (en, ru, th). Нельзя добавить только на одном языке и "потом переведу".

Файл: `src/i18n/translations.js` — три блока (en, th, ru).

## I18N-R2. Формат ключа

Ключ перевода — на английском, в camelCase. Например: `bookingTotalPrice`, `propResponsibleLabel`.

Не `booking_total_price`, не `Booking Total Price`, не на русском.

## I18N-R3. Не создавать дубли

Перед добавлением нового ключа — поиск по файлу переводов. Может такая строка уже есть. У нас есть проблема с дублями (TD-004): `ownerCommissionOneTime` vs
`bookingOwnerCommOnce` — одно и то же, два ключа.

**Правило:** сначала ищи, потом создавай.
