# Сквозные правила: Validation (Проверка данных)

> Эти правила действуют во ВСЕХ модулях проекта.

## VL-1. Проверка на клиенте

Обязательные поля проверяются **на клиенте** перед отправкой. Пользователь не должен ждать ответ сервера чтобы узнать что забыл заполнить имя.

## VL-2. Дублирование в базе данных

Критичные ограничения **дублируются в базе данных** (CHECK constraints). Клиентская проверка — для удобства пользователя. Серверная — для защиты данных.

Если клиентская проверка сломается — база не пропустит плохие данные.

## VL-3. Одинаково на обеих платформах

Проверки должны быть одинаковые на вебе и мобильном. Нельзя: на вебе проверять три поля, а на мобильном одно.

Пример найденной проблемы: локация обязательна на вебе, не обязательна на мобильном (TD-044).
