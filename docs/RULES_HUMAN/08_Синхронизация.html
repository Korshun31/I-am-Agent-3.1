Модуль 8: Синхронизация

I am Agent — Кодекс v2.0
Что делает этот модуль: когда один участник команды меняет данные — другие участники видят изменения. Лёгкий механизм через «рацию», не нагружает сервер.

8.1 Как работает

✅
Участники компании подключены к одному каналу. Когда кто-то изменил данные (создал объект, забронировал, удалил контакт) — отправляется короткий сигнал «данные изменились». Другие участники получают сигнал и обновляют свой экран.
✅
Сигнал не передаёт данные — только говорит «обновись». Это безопасно и лёгко. Сервер почти не замечает.
✅
Пользователь не получает свои собственные сигналы — только чужие. Чтобы экран не обновлялся дважды.

8.2 Тестовая выгрузка данных

✅
Экспериментальная функция (тариф Korshun) — выгрузка всех данных из основной базы в другую. Только на мобильном. Для тестирования и резервного копирования. Будет доработана в будущем.

Проблем в этом модуле нет. Всё работает одинаково на обеих платформах.
