Модуль 6: Календарь

I am Agent — Кодекс v2.0
Что делает этот модуль: личный календарь пользователя — создание событий (встречи, задачи, напоминания), повторяющиеся события, напоминания. Не путать с бронированиями — это отдельный модуль.

6.1 Создание события

✅
Обязательное поле: Название. Дата — из выбранной на календаре. Остальное по желанию: время, цвет, комментарии, напоминания, повтор.
✅
Напоминания: в момент события, за 5/10/15/30 минут, за 1/2 часа, за 1 день, за 1 неделю, за 1 месяц.
✅
Повтор: ежедневно, еженедельно, ежемесячно, ежегодно. Повторяющееся событие показывается на всех подходящих датах.

6.2 Редактирование и удаление

✅
Каждый пользователь редактирует и удаляет только свои события. Подтверждение перед удалением.
✅
Можно пометить событие «выполнено» прямо на экране календаря, без открытия формы.
⚠️TD-110
Нельзя перенести событие на другую дату. Чтобы перенести — нужно удалить и создать заново. Нужно: кликабельная дата → выбор новой даты.

6.3 Видимость

✅
Календарь — личный. Каждый видит только свои события. Админ и агент — одинаковые возможности.
⚠️TD-109
В базе данных политика доступа слишком широкая — агент может прочитать чужие события через прямой запрос. В интерфейсе это не видно, но база не защищена.
✅
На экране календаря отображаются вместе: личные события, заселения/выселения (из бронирований), комиссии собственнику. Визуально различаются цветами и иконками.
