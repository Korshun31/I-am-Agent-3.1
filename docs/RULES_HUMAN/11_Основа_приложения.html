Модуль 11: Основа приложения

I am Agent — Кодекс v2.0
Что делает этот модуль: «скелет» приложения — что происходит при запуске, как определяется платформа, как загружаются данные.

11.1 Запуск приложения

✅
Порядок при запуске:
  1. Показывается экран загрузки (логотип + прогресс)
  2. Проверяется: входил ли пользователь ранее?
  3. Если да → загрузка профиля → рабочий экран
  4. Если нет → экран входа
✅TD-019
Закрыт 2026-04-30. На вебе теперь показывается тот же экран загрузки что и на мобильном — без мелькания формы входа при перезагрузке.

11.2 Разделение платформ

✅
После входа:
  • Телефон → вкладки внизу экрана. Данные загружаются один раз при старте и хранятся в памяти. Переключение между вкладками — мгновенное.
  • Компьютер → боковое меню. Данные загружаются на каждом экране отдельно. Мощность компьютера позволяет.
✅
Это осознанное решение: на телефоне хранилище данных нужно потому что телефон слабее. На компьютере — не нужно потому что мощности хватает.

11.3 Загрузка данных (телефон)

✅
При запуске загружаются четыре набора данных: объекты, бронирования, контакты, события календаря. Прогресс: 4 × 25%. Экран загрузки показывает процент.

11.4 Защита от ошибок

✅
Приложение обёрнуто в защитный слой: если какой-то экран сломается с ошибкой — пользователь увидит сообщение об ошибке, а не белый экран.

11.5 Автоматический выход

✅
Если сессия истекла (давно не открывали приложение) — автоматический переход на экран входа.
