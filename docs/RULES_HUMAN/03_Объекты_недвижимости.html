Модуль 3: Объекты недвижимости

I am Agent — Кодекс v2.0
Что делает этот модуль: управление объектами недвижимости — дома, резорты, кондо. Создание, редактирование, модерация, удаление, фотографии, локации, назначение ответственного.

3.1 Типы объектов

Пять типов, два уровня:

Самостоятельные объекты:

— Дом — отдельный объект, не входит в комплекс
— Резорт — комплекс, контейнер для домов
— Кондо — здание, контейнер для квартир

Дочерние объекты (внутри контейнера):

— Дом в резорте — дом внутри конкретного резорта
— Квартира в кондо — квартира внутри конкретного кондо

✅
Тип объекта задаётся при создании и не может быть изменён. Дом не станет резортом, квартира не станет домом.
✅
Дочерний объект не может существовать без родителя. При удалении резорта/кондо — все дома/квартиры внутри удаляются автоматически. Пользователю показывается двойное предупреждение.
✅
Дочерние объекты наследуют от родителя: город, район, собственника, ответственного агента (отображаются как неизменяемые).

3.2 Создание объекта

✅
Обязательные поля: Название, Город (локация), Район. Всё остальное — по желанию, можно дозаполнить позже. Это важно для агентов «в поле» — быстро зафиксировать объект и уехать.
✅
Если создаёт админ — объект сразу виден в списке (статус «одобрен»). Если создаёт агент — объект на проверке (статус «ожидает»), админ должен одобрить.
✅
Агент может создать объект только если у него включён переключатель «Добавление объектов».

Шаги заполнения на мобильном (пошаговый визард):

Для дома / дома в резорте / квартиры — 8 шагов: Информация → Характеристики → Описание → Фото и видео → Удобства → Дополнительно → Цены и коммуналка → Комментарии.

Для резорта / кондо — 5 шагов (без удобств, доп. настроек и цен — контейнеры не сдаются).

На вебе — боковая панель с вкладками: Основное, Цены, Коммуналка, Удобства, Фото.

✅
Поле «Собственник» — видно и админу, и агенту. Поле «Ответственный агент» — видно только админу. Агент не видит и не может менять ответственного.

3.3 Редактирование объекта

✅
Админ редактирует всё (кроме типа) и сохраняет сразу. Без модерации.
✅
Агент — два независимых переключателя:
— «Редактирование информации» — если включён, информация сохраняется напрямую
— «Редактирование цен» — если включён, цены сохраняются напрямую

Если переключатель выключен — изменения идут через черновик на проверку админу. Если оба выключены — кнопка «Редактировать» не видна.
✅
Агент видит все поля при редактировании (кроме «Ответственный»). Может менять любые. При сохранении система разделяет: разрешённые → напрямую, неразрешённые → черновик.
⚠️TD-058
Раздельное сохранение (разрешённое напрямую, неразрешённое через черновик) не реализовано. Сейчас — всё или ничего.
✅
Отклонённый объект — всегда через черновик, даже если оба переключателя включены.

3.4 Модерация (одобрение / отклонение)

✅
Два типа модерации:
1. Новый объект — агент создал → статус «ожидает» → админ одобряет или отклоняет
2. Черновик изменений — агент отредактировал → черновик → админ одобряет (изменения применяются) или отклоняет (оригинал не меняется)
✅
При отклонении админ обязан указать причину. Причина отправляется агенту в уведомлении и сохраняется в журнале отклонений.
✅
Админ находит объекты на проверке тремя способами: значок ⏳ в списке, колокольчик уведомлений, фильтр «На проверке».
⚠️TD-055
Админ не видит сравнение «было / стало» при просмотре черновика. Нужно: изменённые поля выделены, старое и новое значение рядом.

3.5 Удаление объекта

✅
Админ может удалить любой объект.
✅
Агент может удалить объект только если: (1) сам создал и (2) объект не одобрен. Одобренный — не может, даже если сам создавал.
✅
При удалении объекта автоматически удаляются: дочерние объекты, все бронирования, уведомления, черновики, история отклонений. Контакты собственника не удаляются.
⚠️TD-061
При удалении объекта с бронированиями нет предупреждения. Нужно: «У этого объекта N бронирований. Они тоже будут удалены.»

3.6 Фотографии

✅
Лимит фото: Standard — 10, Premium/Korshun — 30.
✅
Два размера: 1200px (полный, для просмотра) и 150px (миниатюра, для списков и карточек). JPEG, качество 0.85.
⚠️TD-064
Миниатюры не генерируются. Везде загружается полноразмерное фото даже для карточки 64px.
⚠️TD-053
При удалении объекта фотографии остаются в хранилище. Утечка места.

3.7 Локации и районы

✅
Локация = Страна / Регион / Город. Все три обязательны. Выбираются из справочника (выпадающие списки).
✅
У каждой локации могут быть районы — более точное разделение внутри города. При переименовании или удалении района — каскадно обновляется на всех объектах.
⚠️TD-067
Удаление локации не проверяет привязанные объекты. Нужно блокировать: «У этой локации N объектов.»

3.8 Ответственный агент

✅
Ответственный агент — определяет кто видит объект. Если не назначен — ответственный «Компания» (админ). При смене ответственного на резорте/кондо — каскадно обновляется на всех дочерних.
✅
Информация об ответственном — только для админа. Агент не видит кто ответственный.

3.9 Видимость объектов

✅
Админ видит все объекты компании. Агент видит только объекты где он ответственный. Объекты «компании» (без ответственного) — агент не видит.
✅
По умолчанию список показывает только одобренные. Ожидающие и отклонённые — через фильтр «На проверке».
