Модуль 1: Регистрация и вход

I am Agent — Кодекс v2.0
Что делает этот модуль: позволяет создать аккаунт, войти в систему, восстановить пароль, управлять сессией и профилем.

1.1 Регистрация нового пользователя

✅
Для регистрации нужны: электронная почта и пароль. Имя — по желанию. Минимум полей для быстрой регистрации.
🔴TD-031
Пароль должен быть не менее 8 символов. Пароли из списка самых популярных («123456», «password», «qwerty») должны отклоняться.

Сейчас: допускается 6 символов, любой пароль.
✅
Пароль и подтверждение пароля должны совпадать.
✅TD-015
Закрыт 2026-04-30. После регистрации показывается экран «Проверьте почту» с email-ом юзера. По клику на ссылку из письма — открывается экран «Почта подтверждена» с инструкциями для веба и телефона. Без подтверждения вход не пускает — выдаётся понятное сообщение. На мобильном пока ссылка открывается в браузере — для нативного открытия в приложении заведена отдельная задача TD-118.
⚠️TD-016
Регистрация с одноразовых почтовых адресов (mailinator.com, tempmail.com) должна быть запрещена — защита от ботов.

Не реализовано.
✅
При регистрации автоматически создаётся: учётная запись, профиль, Workspace (невидимый контейнер для данных) и запись о том что пользователь — администратор своего workspace. Всё мгновенно и незаметно.
✅
При регистрации по приглашению — workspace НЕ создаётся. Агент работает только с данными компании которая его пригласила. Это обеспечивается «умным триггером» в базе данных.
⚠️TD-040
Если пользователь пытается зарегистрироваться, а на его почту уже есть приглашение — должно появиться окно: «Компания X вас пригласила. Принять / Отклонить». Принял — ввод кода, регистрация как агент. Отклонил — обычная регистрация с workspace.

Не реализовано.
✅
Ошибки регистрации показываются прямо на экране под кнопкой (красный текст), не во всплывающем окне.

1.2 Вход в систему

✅
Для входа нужны: email и пароль. При неправильном пароле — понятное сообщение на языке пользователя.
🔴TD-032
После 5 неудачных попыток ввода пароля — вход должен блокироваться на 15-30 минут. Защита от подбора пароля.

Сейчас: бесконечные попытки.
✅TD-039
Закрыт 2026-04-30. Кнопка «Войти» теперь показывает текст «Сохранение…» и блокируется на время проверки пароля — нельзя нажать повторно. Паритет с экраном регистрации.
✅
Клавиатурная навигация: Enter в поле email → фокус на пароль. Enter в поле пароль → отправка формы.

1.3 Вход через Google и Facebook

⚠️
Кнопки Google и Facebook временно скрыты до завершения проверки безопасности. Функции работают, просто не видны.
✅TD-033
Снят 2026-04-30. OAuth-кнопки (вход через Google/Facebook) уходят из продукта — настраивать безопасный протокол PKCE для функции которая будет удалена нет смысла. Чистка остатков OAuth-кода — отдельный пункт TD-116.
⚠️TD-041
Когда OAuth будет включён — вход через Google/Facebook должен тоже проверять приглашения (как обычная регистрация).

Не критично пока кнопки скрыты.

1.4 Восстановление сессии

✅
Если пользователь уже входил и не выходил — при открытии приложения попадает сразу на рабочий экран. Без повторного ввода пароля.
✅TD-019
Закрыт 2026-04-30. На вебе при перезагрузке страницы теперь показывается тот же экран загрузки что и на мобильном — без мелькания формы входа.

1.5 Восстановление пароля

✅TD-014
Закрыт 2026-04-30. Под кнопкой входа добавлена ссылка «Забыли пароль?». Нажимаешь → вводишь email → получаешь письмо со ссылкой → клик переводит на экран установки нового пароля → после сохранения возврат к экрану входа. На телефоне ссылка открывается в браузере, новый пароль задаётся через веб.

1.6 Смена пароля

✅
Доступна только для пользователей с email (не через Google/Facebook). Для смены нужно ввести текущий пароль и новый.

1.7 Выход и безопасность

✅
При выходе сессия завершается, показывается экран входа. Кэш не очищается — при повторном входе данные загрузятся быстрее.
⚠️TD-037
Должна быть кнопка «Выйти со всех устройств». Если потерял телефон — одним нажатием закрыть все сессии.

Не реализовано.

1.8 Удаление аккаунта

🔴TD-038
Пользователь должен иметь возможность удалить свой аккаунт. Это требование Apple App Store (обязательно для публикации), GDPR (Европа) и PDPA (Таиланд — целевой рынок).

Сейчас: невозможно удалить аккаунт.

1.9 Профиль пользователя

✅
При каждом входе система собирает полный профиль: тариф, роль, режим работы, права агента, язык, валюта. Это определяет что видит и может делать пользователь.
💡TD-035
Профиль собирается из 4-5 запросов к базе. Можно ускорить в 3-5 раз объединив в один запрос.
