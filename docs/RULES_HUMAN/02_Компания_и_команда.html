Модуль 2: Компания и команда

I am Agent — Кодекс v2.0
Что делает этот модуль: позволяет перейти из режима «частный предприниматель» в режим «компания» — дать workspace название и логотип, пригласить сотрудников, управлять их правами.

2.1 Workspace и компания

✅
При регистрации каждому создаётся невидимый workspace — контейнер для данных. Когда пользователь решает работать как компания — workspace получает «вывеску»: название, логотип, контакты. Технически это одна и та же запись, просто с заполненными полями.
✅
Переключение в режим «компания» доступно только на тарифе Premium или Korshun. Для Standard — предложение обновить тариф.
✅
Название компании — обязательное. Остальное (телефон, email, логотип, Telegram, WhatsApp, Instagram, часы работы) — по желанию.
⚠️TD-029
Нет проверки названия — можно создать с пустым именем или спецсимволами. Нужно: от 2 до 100 символов.

2.2 Деактивация компании

✅
Деактивация запрещена если в команде есть активные агенты. Сначала — убрать всех агентов.
✅
При деактивации: приглашения отзываются, компания переходит в «неактивна». Данные не удаляются — можно вернуться в любой момент.

2.3 Приглашение сотрудника

✅
Двойная проверка безопасности: для вступления нужны ссылка (токен) и 6-значный код. Одной ссылки недостаточно.
✅
Нельзя пригласить email который уже зарегистрирован в системе. Один email = один аккаунт.
✅
Приглашение действительно 7 дней.
🔴TD-027
Нет ограничения на попытки ввода кода — можно перебрать за минуты. Нужно: максимум 5 попыток, потом приглашение отзывается.

Не реализовано.

2.4 Принятие приглашения

Путь A — через ссылку:

  1. Открывает ссылку → вводит 6-значный код
  2. Если email уже в системе → «Используйте другой email»
  3. Если email свободен → заполняет имя, пароль → регистрируется как агент
  4. Админу приходит уведомление «принял приглашение»

Путь B — через обычную регистрацию:

  1. Заполняет форму, нажимает «Создать аккаунт»
  2. Если на этот email есть приглашение → модальное окно: «Компания X вас пригласила. Принять / Отклонить»
  3. Принять → ввод кода → регистрация как агент
  4. Отклонить → приглашение отменяется → обычная регистрация
  5. Админу уведомление в обоих случаях
✅
При вступлении пользователь всегда получает роль агент. Нельзя присоединиться как админ.
✅
Один email = один аккаунт. Нельзя быть и владельцем workspace, и агентом в чужой команде одновременно.
⚠️TD-021
Принятие приглашения работает только на вебе. На мобильном экрана нет.

2.5 Права агента

Два типа ограничений: переключатели (админ настраивает) и жёсткие правила (нельзя изменить).

Настраиваемые переключатели

РазрешениеЧто даёт агенту
Управлять объектамиСоздаёт, редактирует (включая цены) и удаляет свои объекты. Объекты публикуются сразу, без проверки админом.
Управлять бронированиямиСоздаёт, редактирует и удаляет свои бронирования.
✅
С 2026-04-30 (упрощение прав) — модерации объектов админом больше нет, все новые объекты публикуются сразу. Финансы своих бронирований и работа со своими контактами всегда доступны агенту без отдельных переключателей.

Жёсткие правила (нельзя изменить)

Агент никогда не может:

— Видеть или менять ответственного за объект/бронирование
— Редактировать/удалять чужие объекты и бронирования
— Управлять локациями (только читать + добавлять районы)
— Приглашать или удалять членов команды
— Менять права других агентов
— Видеть чужие объекты и бронирования

2.6 Удаление агента из команды

✅
При удалении агента:
  1. Запись помечается как «неактивна» (история сохраняется)
  2. Email подменяется на системный — освобождается для нового аккаунта
  3. Аккаунт блокируется — вход невозможен
  4. Снимается доступ к локациям
  5. Объекты переходят «к компании» (ответственный обнуляется)
  6. Бронирования переходят «к компании»
✅
Аккаунт не удаляется из системы. ID сохраняется — вся история (кто что создал) остаётся.
✅
Освобождённый email можно использовать для новой регистрации или нового приглашения — с новым ID.
⚠️TD-042
Текущая реализация просто удаляет запись вместо пометки «неактивна» и не освобождает email. Нужно переписать.

2.7 Управление командой на разных платформах

⚠️TD-022
Управление командой работает только на вебе. В мобильном приложении можно только переключить режим и заполнить данные компании. Раздел «Команда» отсутствует.

2.8 Безопасность

🔴
Подтверждение email (из модуля 1, TD-015) критично для приглашений. Без подтверждения злоумышленник может зарегистрироваться с чужим email и принять чужое приглашение.
⚠️TD-030
Нет журнала действий с командой: кто кого пригласил, кто принял, кто был удалён. При спорах — нет доказательств.
