## 6.1 Review-flow (актуальный)

| Уведомление | Кнопка | Открывает | Действия |
|------------|--------|-----------|---------|
| `property_submitted` | 🔍 Посмотреть и принять решение | Правая панель (`WebPropertyEditPanel`, readOnly) | Одобрить / Отклонить |
| `edit_submitted` | 🔍 Посмотреть и принять решение | Правая панель (merged preview) | Одобрить / Отклонить |
| `property_approved` | — | — | Агент видит статус |
| `property_rejected` | — | — | Агент видит статус + история |

> **Важно:** `DiffModal` сохранён в коде, но **не используется** в review-flow. Обе проверки идут через правую панель.

## 6.2 Статусы объекта

| Статус | ⏳ в левой карточке | Правая панель (Admin) | Правая панель (Agent) |
|--------|--------------------|-----------------------|-----------------------|
| `pending` | ✅ | Approve / Reject block | "На проверке" badge (без Edit) |
| `approved` | ❌ | Стандартный вид | Стандартный вид |
| `rejected` | ✅ | "На обработке" + Edit + История | "На обработке" + Edit + История |

## 6.3 История отклонений

- Блок **"История отклонений"** показывается в `PropertyDetail` при `rejected` или при наличии legacy `rejection_reason`.
- Нумерованный список: #1 (первое) → #N (последнее). Последнее — жирное/красное, предыдущие — приглушённые.
- История хранится в таблице `property_rejection_history` (append-only).
- При auto-approve (admin edit + save) история **не удаляется**, блок скрывается только в `approved` состоянии.
