# Дизайн-система мобильного приложения «I am Agent»

> Единственный источник правды по визуальному оформлению мобильного приложения.
> Если что-то противоречит этому документу — правится этот документ или код, но не оба расходятся.
>
> Поддерживается на ветке `design/mobile-ui-refresh`. Дата создания: 2026-05-12.

---

## 0. Идея дизайна одной фразой

Спокойный, светлый, мягкий интерфейс в духе iOS. Без сплошных тяжёлых заливок, без агрессивных контрастов. Акцент — приглушённо-бирюзовый. Иерархия строится тонко: размером шрифта, его весом и оттенком серого, а не цветом и не толстой рамкой. Цвет — функциональный (статус, отметить категорию), не декоративный.

## 3. Отступы и ритмика

### 3.1 Внутри форм (между полями)

- Метка → её поле: **6pt** (тесная связь, видно что подпись от поля).
- Блок «метка+поле» → следующий блок: **16pt** (читаемая пауза между разными полями).

Композитные поля (две части в одной строке) оборачиваются в общий контейнер с нижним отступом 16, внутренние элементы — без своих нижних отступов.

### 3.2 Между крупными элементами экрана

| Что                                | Отступ |
|------------------------------------|--------|
| Левый/правый край экрана           | 20     |
| Шапка → следующий блок             | 14     |
| Ряд кнопок-действий → контент      | 14     |
| Между секциями карточки            | 16–20  |
| Внутри карточки секции             | 12–16  |
| Между точками прогресса визарда    | 6      |

ТОП значений отступов, реально применяемых в проекте: 4, 6, 8, 10, 12, 14, 16, 20, 24. Из них самые частые — 12, 14, 16, 20.

## 5. Тени

В мобильном приложении используются три уровня тени.

**Лёгкая (для карточек на белом):**
```
shadowColor: '#000', shadowOffset: { width: 0, height: 2 },
shadowOpacity: 0.04, shadowRadius: 8, elevation: 1
```

**Средняя (для активных карточек, плавающих кнопок):**
```
shadowColor: '#000', shadowOffset: { width: 0, height: 2 },
shadowOpacity: 0.08, shadowRadius: 12, elevation: 2
```

**Глубокая (для модалки поверх контента):**
```
shadowColor: '#000', shadowOffset: { width: 0, height: 4 },
shadowOpacity: 0.2, shadowRadius: 12, elevation: 8
```

Другие комбинации — не использовать. Если кажется что нужна «ещё одна» тень — скорее всего нужно поднять элемент в иерархии (например, использовать модалку вместо карточки).

## 8. Кнопки и интерактив

### 8.1 Главная кнопка действия

Контурная, не залитая. Полупрозрачный фон акцента, контур акцента, текст акцента.

```
backgroundColor: rgba(61,125,130,0.06)
borderWidth: 1
borderColor: rgba(61,125,130,0.5)
borderRadius: 12
paddingVertical: 13, paddingHorizontal: 14
minHeight: 46
текст: 15 / 600 / #3D7D82
```

### 8.2 Кнопка «Далее / Сохранить» в визарде

Контурная, без фона, текст и контур — акцент. Чуть толще бордер.

```
backgroundColor: transparent
borderWidth: 1.5
borderColor: #3D7D82
borderRadius: 12
текст: 15 / 600 / #3D7D82, со стрелкой ›
```

### 8.3 Кнопка «Назад» в визарде

Без фона, без рамки. Серый текст с типографской стрелкой `‹  Назад`.

```
color: #6B6B6B, без фона и бордера
```

### 8.4 Сегментированный тогглер ($/% и аналоги)

Общий контур, активный сегмент — полупрозрачный акцент + текст акцента.

```
общий контейнер: borderRadius 7, borderWidth 1, borderColor #E9ECEF
активный сегмент: backgroundColor rgba(61,125,130,0.06), текст #3D7D82
неактивный сегмент: backgroundColor #F7F7F9, текст #666
сегмент: paddingHorizontal 12, paddingVertical 13
```

### 8.5 Нативный переключатель (Switch)

Единственный элемент со сплошной заливкой акцента.

```
trackColor: { false: #D1D1D6, true: #3D7D82 }
thumbColor: #FFFFFF
ios_backgroundColor: #D1D1D6
```

### 8.6 Чекбокс

Через общий компонент `src/components/Checkbox.js`.

## 9. Модалки

### 9.1 Главная модалка-визард

Открывается через общий каркас `src/components/ModalScrollFrame.js`. Скруглённая белая карточка по центру экрана, максимум 380 ширина.

- Фон карточки: `#FFFFFF` (сплошной, не полупрозрачный).
- Радиус: задаёт ModalScrollFrame.
- Тень: глубокая (см. раздел 5).
- Шапка внутри: заголовок 20 / 600 / -0.3, справа крестик 22pt, оба не масштабируются.
- Под шапкой — ряд точек прогресса.
- Подвал — `WizardFooter` с кнопками «Назад» / «Далее / Сохранить», верхний бордер `rgba(0,0,0,0.07)`.

### 9.2 Подмодалка-пикер (поверх модалки)

Полупрозрачная белая коробка с блюром поверх затемнённого фона.

- Фон коробки: `rgba(255,255,255,0.72)` + `BlurView`.
- Бордер: `rgba(255,255,255,0.5)`, толщина 1.
- Радиус: 16.
- Максимум: ширина 400, высота 80% экрана.
- Тень: глубокая.
- Заголовок мельче, чем у модалки-визарда: 16 / 600.
- Нижняя граница шапки: `rgba(0,0,0,0.05)`.

## 10. Шапка экрана

- Контейнер: `paddingHorizontal: 20, marginBottom: 14, paddingTop: статус-бар + 12`.
- Слева: кнопка «назад» 36×36 со стрелкой `chevron-back` 20pt `#2C2C2C`.
- В центре: заголовок 20 / 600 / -0.3 / `#2C2C2C`.
- Справа: либо пустой блок 36×36 (для симметрии), либо кнопки действий.

Ряд кнопок-действий под шапкой:
- `flexDirection: row, justifyContent: space-between, marginBottom: 14, paddingHorizontal: 20`.
- Слева — обычно «удалить» (иконка корзины 22pt `#888`).
- Справа — группа иконок, между ними gap 12.

## 11. Состояния элементов

- **Отключено (disabled):** уменьшить непрозрачность всего элемента до `0.4`. Цвет и шрифт не меняются.
- **Загрузка вместо текста кнопки:** заменить надпись на `ActivityIndicator` (не показывать «…»).
- **Пустой экран / пустой список:** монохромная иконка-контур (32pt `#999`), под ней метка 14 / 400 / `#6B6B6B`, ниже — кнопка действия по шаблону раздела 8.1.
- **Фокус поля ввода / нажатие на кнопку:** отдельно не оформляется, отрисовывает система.

## 12. Адаптивность

Используется утилита `sz()` из `src/utils/scale.js`: масштабирует размер относительно референсного айфона (440pt по ширине), вниз — до коэффициента 0.85 (iPhone SE и подобные).

**Что оборачивать в `sz()`:**
- В **модалках-визардах** — почти всё внутри: `fontSize` (кроме фиксированных заголовков 20pt и 16pt), `padding*`, `margin*`, `borderRadius`, `gap*`, `width`, `height`, `borderWidth*`.

**Что НЕ оборачивать:**
- На основных экранах (не модалках) — числа из этой системы дизайна берутся фиксированные.
- Цвета, opacity, flex, zIndex, проценты.
- Заголовок экрана (20pt) и заголовок подмодалки (16pt) — фиксированные.
- Шкалу FONT и WEIGHT — никогда.
- Нативные элементы (Switch) — масштабируются через `transform: [{ scale: SCALE }]`.

## 13. Анимации

Общее правило: переходы и движения — короткие и сдержанные.

- Длительность для микро-анимаций (открытие подмодалки, появление точки прогресса): 200 мс.
- Длительность для крупных переходов (модалка с экрана, смена шага визарда): 300 мс.
- Easing: системный по умолчанию (без рывков и баунса).
- Большая нативная анимация в проекте — синхронный горизонтальный скролл двух колонок в календаре броней (используется `Animated.event` с `useNativeDriver: true`). Любой новый параллельный скролл — по той же схеме.

## 15. Запреты (антипаттерны)

- Сплошная заливка `#3D7D82` с белым текстом для главной кнопки или выбранного сегмента — даёт «тяжёлый» вид (исключение только Switch).
- Промежуточные размеры шрифта: 9, 10, 11, 13, 15, 17, 18, 19, 24+. Только шкала FONT и фиксированные заголовки 20/16.
- Веса 300, 500, 800. Только 400 / 600 / 700.
- Промежуточные радиусы (11, 13, 15, 18) — только значения из раздела 4.
- Кнопка с текстом «…» во время загрузки — заменить на спиннер.
- Текстовый символ «‹» вместо иконки `chevron-back` в шапке экрана.
- Цветные PNG-картинки в заголовках секций — только монохромные иконки-контуры.
- Двойной нижний отступ в композитных полях (внутренний `marginBottom` обнуляется).
- Тени с произвольными параметрами — только три уровня из раздела 5.
- Глобально оборачивать все `fontSize` в `sz()` на основных (не модальных) экранах.

## 17. Как использовать этот документ

- Перед началом любой дизайн-задачи — открыть этот файл и проверить, какие цвета, размеры и отступы уже есть для этого случая.
- При появлении нового элемента, которого нет в системе, — сначала дописать его сюда, согласовать, и только потом ставить в код.
- При расхождении документа и кода — победитель документ, код подгоняется под него (за исключением раздела 16 — там сначала идёт миграция, потом обновление статуса в этом документе).
