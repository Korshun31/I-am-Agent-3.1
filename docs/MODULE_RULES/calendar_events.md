# Module: Calendar Events

## Скоуп

Модуль покрывает: создание, редактирование и удаление **ручных** событий календаря (встречи, напоминания, задачи). Повторяющиеся события. Напоминания (локальные уведомления).
Toggle "выполнено".

**НЕ покрывается (другие подмодули отображаются на том же экране, но не хранятся в `calendar_events`):**
- Заселения и выселения — генерируются из `bookings`
- Платежи комиссий собственнику — генерируются из `commissionRemindersService`

**Файлы модуля:**
- `src/services/calendarEventsService.js` — CRUD, recurring logic (`eventOccursOnDate`)
- `src/services/calendarRemindersService.js` — локальные уведомления (expo-notifications)
- `src/components/AddCalendarEventModal.js` — создание/редактирование (мобильный)
- `src/screens/AgentCalendarScreen.js` — экран календаря (мобильный)
- `src/web/components/WebAddCalendarEventModal.js` — создание/редактирование (веб)

**DB:**
- FK `calendar_events_agent_id_fkey` → `auth.users(id)` ON DELETE CASCADE (legacy название)
- FK `calendar_events_company_id_fkey` → `companies(id)` ON DELETE SET NULL
- Нет триггеров, нет CHECK constraints
- Календарь — **личный** инструмент. Каждый пользователь видит и управляет только своими событиями.

## Подмодуль 1: Создание события (CE-CR)

### Обязательные поля

**CE-CR-1.** Обязательное поле: **Название**. Проверяется на клиенте (обе платформы).

**CE-CR-2.** Дата события: приходит из контекста экрана (выбранная дата в календаре). По умолчанию — сегодня.

### Поля события

**CE-CR-3.** Поля:
- Название (обязательное)
- Дата (обязательная, из контекста)
- Время (опциональное, формат HH:mm)
- Цвет (из палитры 7 цветов, по умолчанию первый)
- Комментарии (опциональное)
- Напоминания (`reminder_minutes`, JSONB массив) — допустимые значения: 0, 5, 10, 15, 30, 60, 120, 1440, 10080, 43200 минут
- Повтор (`repeat_type`) — `null`, `daily`, `weekly`, `monthly`, `yearly`
- Выполнено (`is_completed`, boolean, по умолчанию false)

### Автоматические поля

**CE-CR-4.** При создании:
- `user_id` = создатель
- `company_id` = из `resolveCompanyId()`

### Напоминания

**CE-CR-5.** При создании с напоминаниями — регистрируются локальные уведомления через `scheduleReminder()`. Формат trigger: `{ type: SchedulableTriggerInputTypes.DATE, date:
triggerDate }` (исправлен в коммите `e35c7a9`).

**CE-CR-5.1.** Если время напоминания уже в прошлом — пропускается без ошибки.

**CE-CR-5.2.** Если разрешения на уведомления не даны — событие сохраняется, уведомления не регистрируются (без ошибки пользователю).

### Сравнение веб vs мобильный

| Аспект | Мобильный | Веб | Совпадает? |
|---|---|---|---|
| Валидация названия | ✅ | ✅ | ✅ |
| Поля | ✅ | ✅ | ✅ |
| Пикер напоминаний | ✅ Модалка с чекбоксами | ✅ Выпадающий с чекбоксами | ✅ |
| Пикер повтора | ✅ Модалка | ✅ Выпадающий | ✅ |
| Регистрация уведомлений | ✅ `scheduleReminder` | ❌ Нет | ⚠️  TD-108 |

**Все правила применяются одинаково на вебе и мобильном.**

## Подмодуль 2: Редактирование и удаление (CE-ED)

### Редактирование

**CE-ED-1.** Пользователь может редактировать **только свои** события. RLS: `own update in company` (`user_id = auth.uid()`).

**CE-ED-2.** При редактировании можно изменить все поля кроме даты.

**CE-ED-2.1.** *(TD-110)* Перенос события на другую дату сейчас **невозможен** — дата зафиксирована. Предлагаемое решение: кликабельный заголовок с датой → date picker. Обе
платформы.

**CE-ED-3.** Toggle "выполнено" — можно переключить прямо на экране календаря, без открытия формы редактирования.

**CE-ED-4.** При редактировании: старые уведомления отменяются (`cancelReminders`), новые регистрируются.

### Удаление

**CE-ED-5.** Пользователь может удалить **только свои** события. RLS: `own delete in company` (`user_id = auth.uid()`).

**CE-ED-6.** Подтверждение перед удалением (Alert на мобильном, confirm() на вебе).

**CE-ED-7.** При удалении: уведомления отменяются (`cancelReminders`).

### Сравнение веб vs мобильный

| Аспект | Мобильный | Веб | Совпадает? |
|---|---|---|---|
| Редактирование полей | ✅ | ✅ | ✅ |
| Toggle "выполнено" | ✅ | ✅ | ✅ |
| Перенос на другую дату | ❌ | ❌ | ✅ Одинаково нет (TD-110) |
| Отмена уведомлений при редактировании | ✅ `cancelReminders` | ❌ Нет | ⚠️  TD-108 |
| Отмена уведомлений при удалении | ✅ `cancelReminders` | ❌ Нет | ⚠️  TD-108 |
| Подтверждение удаления | ✅ Alert | ✅ confirm() | ✅ |

## Подмодуль 3: Видимость и отображение (CE-VIS)

### Личный календарь

**CE-VIS-1.** Календарь — **личный** инструмент. Каждый пользователь (админ или агент) видит и управляет **только своими** событиями.

**CE-VIS-1.1.** *(TD-109)* RLS политика `company member read` даёт доступ ко всем событиям компании — слишком широкая. JS-код фильтрует по `user_id`, но БД не защищена. Нужно
заменить на `user_id = auth.uid()`.

### Recurring events (повторяющиеся)

**CE-VIS-2.** Повторяющиеся события отображаются на всех подходящих датах. Логика `eventOccursOnDate()`:
- `daily` — каждый день начиная с даты создания
- `weekly` — каждую неделю в тот же день недели
- `monthly` — каждый месяц в то же число
- `yearly` — каждый год в ту же дату

Событие не отображается на датах **до** даты создания.

### Экран календаря

**CE-VIS-3.** На экране календаря отображаются вместе:
- Ручные события пользователя (из `calendar_events`)
- Заселения и выселения (из `bookings`) — автоматически
- Комиссии собственнику (из `commissionRemindersService`) — автоматически

Ручные и автоматические события визуально различаются (разные цвета и иконки).

### Сравнение веб vs мобильный

| Аспект | Мобильный | Веб | Совпадает? |
|---|---|---|---|
| Личный календарь | ✅ JS-фильтр `user_id` | ✅ JS-фильтр `user_id` | ✅ |
| Recurring events | ✅ `eventOccursOnDate` | ✅ (общий код) | ✅ |
| Ручные + авто события вместе | ✅ `AgentCalendarScreen` | ✅ Веб-календарь | ✅ |

## Связь с RBAC

- Нет permissions для календаря — каждый работает только со своими событиями
- Нет ролевых различий — админ и агент имеют одинаковые возможности

## Связь с другими модулями

| Модуль | Связь |
|---|---|
| **Bookings** | Заселения/выселения отображаются на экране календаря (генерируются из bookings, не хранятся в calendar_events) |
| **Notifications & Reminders** | `calendarRemindersService` регистрирует локальные уведомления для событий |
