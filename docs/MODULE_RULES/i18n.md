# Module: i18n & Settings

## Скоуп

Модуль покрывает: многоязычность (en/ru/th), валюту (THB/USD/EUR/RUB), хранение языковых настроек, функцию перевода, курсы валют для конвертации в статистике (TD-120).

**Файлы модуля:**
- `src/i18n/translations.js` — переводы (3158 строк, три языковых блока) + функции хранения/получения языка
- `src/context/LanguageContext.js` — контекст языка и валюты
- `src/utils/currency.js` — символы валют
- `src/services/currencyRatesService.js` — клиент к публичному `fxratesapi.com` (без ключа, CORS открыт)
- `src/context/CurrencyRatesContext.js` — кэш курсов на сутки в AsyncStorage, индексация в `Map` для O(log n) поиска
- `src/utils/currencyConvert.js` — `convertAmount` / `indexRates` / `bookingFxDate` / `bookingCurrency` (используется только в `statisticsCalc`)

## Языки

**I18N-1.** Три поддерживаемых языка: **en** (английский), **ru** (русский), **th** (тайский). Дефолт: en.

**I18N-2.** Все UI-строки — через функцию `t(key)`. Если ключ не найден в текущем языке → fallback на en → fallback на сам ключ.

**I18N-3.** При добавлении новой UI-строки — **обязательно** добавить во все три языка.

## Валюты

**I18N-4.** Четыре валюты: THB (฿), USD ($), EUR (€), RUB (₽). Дефолт: THB.

**I18N-5.** В обычном UI (формы броней/объектов, списки, карточки) конвертация не выполняется — пользователь вводит и видит суммы в той валюте, в которой они сохранены. Выбранная валюта влияет только на символ.

**I18N-5b.** Исключение — экраны «Статистика» (мобайл `StatisticsScreen` и веб `WebStatisticsScreen`): KPI/графики/топ-объекты/лидерборд агентов пересчитываются в выбранную отчётную валюту через `convertAmount` (TD-120). Курсы тянутся клиентом из `fxratesapi.com` один раз в сутки, кэшируются в AsyncStorage. Дата курса для брони: `checkIn` если бронь уже наступила, иначе `createdAt` (см. `bookingFxDate`). При пустом кэше / отсутствующем курсе работает graceful degrade — amount возвращается без конвертации.

## Хранение

**I18N-6.** Язык:
- Локально: AsyncStorage (мобильный) / localStorage (веб), ключ `@app_language`
- В БД: `users_profile.settings.web_language` (веб) и `settings.app_language` (мобильный)
- При старте: загружается из локального хранилища → применяется. Также синхронизируется с БД (AU-PROFILE-5).

**I18N-7.** Валюта:
- Локально: AsyncStorage / localStorage, ключ `@app_currency`
- В БД: `users_profile.settings.selectedCurrency`

## dayjs

**I18N-8.** dayjs locale: `ru` для русского, `en` для остальных. Тайский locale не подключён — даты на тайском отображаются на английском.

## Сравнение веб vs мобильный

| Аспект | Мобильный | Веб | Совпадает? |
|---|---|---|---|
| Файл переводов | ✅ `translations.js` | ✅ Тот же файл | ✅ |
| LanguageContext | ✅ | ✅ | ✅ |
| Функция t() | ✅ | ✅ | ✅ |
| Хранение языка | AsyncStorage | localStorage (через AsyncStorage shim) | ✅ |
| Переключатель языка | ✅ AccountScreen | ✅ WebAccountScreen | ✅ |
| Переключатель валюты | ✅ | ✅ | ✅ |

Расхождений нет.
