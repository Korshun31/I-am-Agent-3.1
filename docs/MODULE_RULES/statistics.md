# Module: Statistics & Dashboard

## Скоуп

Модуль покрывает: отображение статистики по объектам, бронированиям, заездам, занятости, комиссиям. Веб-дашборд с календарём дня и отдельная веб-страница статистики с графиками.

**Файлы модуля:**
- `src/screens/StatisticsScreen.js` — статистика (мобильный, 341 строка): KPI-карточки, два графика (revenue и bookings), топ-5 объектов, лидерборд агентов, помесячный пикер.
- `src/web/screens/WebStatisticsScreen.js` — отдельная страница статистики (веб, 245 строк): пикер периода, KPI, графики, лидерборд агентов.
- `src/web/screens/WebDashboardScreen.js` — дашборд (веб, 591 строка): календарь дня (заезды/выезды/комиссии/личные события) + дополнительная сводка.
- `src/utils/dashboardStats.js`, `src/utils/statisticsCalc.js` — утилиты подсчётов (унифицированные формулы, TD-121).

## Правила

### Общие

**ST-1.** Статистика строится из данных `getProperties()` и `getBookings()`. Нет отдельных серверных расчётов — всё считается на клиенте.

**ST-2.** Для агента: статистика показывает только его объекты и бронирования (`responsible_agent_id = uid`, `agentId = uid`). Для админа: все данные компании.

**ST-3.** "Клиент собственника" (`notMyCustomer`) исключается из подсчёта бронирований и клиентов.

### Данные статистики

**ST-4.** На обеих платформах должны отображаться:
- Количество объектов по типам (дома, резорты/дома в резорте, кондо/квартиры).
- Бронирования: текущий vs прошлый месяц.
- Заезды: текущий vs прошлый месяц.
- Занятость: мои клиенты / клиенты собственника.
- Ближайшие заезды.
- Комиссии собственнику.
- Топ-5 объектов и лидерборд агентов.

**ST-4.1.** Календарь дня (заезды, выезды, комиссии, личные события) — отдельный блок только на вебе (`WebDashboardScreen`). На мобайле отдельной плитки «календарь дня» нет — данные просматриваются через AgentCalendarScreen и BookingCalendarScreen.

> Старый пункт «Объекты на проверке (pending)» убран: модерация снята в фазе 2 simple-perms (миграция `20260429000000`), все объекты создаются `approved` по DB-default.
