# 🐛 Ошибки и фиксы — 2026-04-10

## Инцидент 1 — Скролл Calendar при появлении объектов через поиск

**Проблема:** При вводе поискового запроса объекты появлялись, но скролл уходил в начало (x=0). Причина: `hasScrolledOnceRef.current = true` (установлен при первом открытии), `timelineScrollXRef.current = 0` (пользователь ещё не скроллил) → else-ветка скроллила к 0.

**Фикс:** В else-ветке добавлена проверка:
```js
const targetX = timelineScrollXRef.current > 0
  ? timelineScrollXRef.current
  : initialScrollX;
```

**Вывод:** ref-переменные нельзя считать "живыми" — нужен явный fallback на семантически корректное значение.

---

## Инцидент 2 — Скролл не восстанавливался при возврате из PropertyDetailScreen

**Проблема:** При нажатии на объект в левой колонке Calendar показывается `PropertyDetailScreen` через условный рендер (`if (selectedPropertyForDetail) return ...`). При возврате (`onBack → setSelectedPropertyForDetail(null)`) скролл не восстанавливался. Существующий useEffect на `selectedBooking` не покрывал этот сценарий.

**Фикс:** Добавлен отдельный useEffect на `selectedPropertyForDetail`:
```js
useEffect(() => {
  if (!selectedPropertyForDetail && rightScrollRef.current) {
    setTimeout(() => { rightScrollRef.current?.scrollTo({ x: targetX }); }, 50);
  }
}, [selectedPropertyForDetail]);
```

**Вывод:** Условный рендер (`if ... return`) — не навигация. ScrollView остаётся смонтированным, но его визуальная позиция сбивается при ре-рендере. Нужен явный восстановительный useEffect.

## Инцидент 3 — Откат MIN_COL_WIDTH = MONTH_WIDTH

**Проблема:** Попытка задать `MIN_COL_WIDTH = MONTH_WIDTH` не сработала визуально так как ожидалось (100 = 100, разницы нет, но семантически зависимость от константы была лишней).

**Фикс:** git revert + явное значение `MIN_COL_WIDTH = 100`.

**Вывод:** Использование константы в качестве значения другой константы оправдано только если между ними есть смысловая связь, а не совпадение числа.

## Инцидент 4 — Keyboard не импортирован

**Проблема:** При добавлении `TouchableWithoutFeedback` + `Keyboard.dismiss` в BookingCalendarScreen.js — ошибка при запуске: `Keyboard` не был импортирован из react-native.

**Фикс:** Добавлен `Keyboard` в импорт отдельным коммитом.

**Вывод:** Добавление новых API из react-native требует явного импорта — линтер не всегда ловит это до запуска.
