# 🧠 Решения за день — 2026-04-10

## Решение 1 — Два режима listToShow в Calendar (поиск vs фильтр)

**Контекст:** `filterFn` возвращает `false` когда фильтр не выбран — это нужно чтобы не показывать 200+ объектов при первом открытии. Но поиск должен работать и без фильтра.

**Проблема:** Единый путь через filterFn делал поиск без фильтра невозможным — все объекты скрывались.

**Решение:** Два явных пути в useMemo:
```js
if (!filterValues && q) {
  // поиск без фильтра — берём все объекты, применяем только текст
} else {
  // фильтр активен — применяем filterFn + текст
}
```

**Почему так:** Разделение обязанностей — filterFn отвечает за фильтрацию по параметрам, текстовый поиск независим от него. Смешивание в одном условии создавало неочевидное поведение.

## Решение 2 — hasScrolledOnceRef вместо сброса позиции

**Контекст:** При каждом появлении вкладки Calendar старый код скроллил к `initialScrollX`. Это правильно при первом открытии, но раздражает при возврате — пользователь терял свою позицию.

**Решение:** `hasScrolledOnceRef` — однократный флаг. Первое появление → `initialScrollX`. Все последующие → `timelineScrollXRef.current` (или `initialScrollX` если не скроллил).

**Почему не useState:** Изменение ref не вызывает ре-рендер — это именно то что нужно для флага навигационного состояния.

## Решение 3 — Три независимых useEffect для восстановления скролла

**Контекст:** Скролл нужно восстанавливать в трёх сценариях: смена вкладки, возврат из карточки брони, возврат из карточки объекта.

**Решение:** Три отдельных useEffect с разными зависимостями:
1. `[effectiveVisible, initialScrollX]` — переключение вкладки
2. `[selectedBooking, initialScrollX]` — возврат из BookingDetailScreen
3. `[selectedPropertyForDetail]` — возврат из PropertyDetailScreen

**Почему не один:** Каждый сценарий имеет разный триггер и разную семантику. Объединение в один useEffect с несколькими условиями создало бы хрупкую логику с побочными эффектами.

## Решение 4 — Адаптивная ширина левой колонки без верхнего ограничения

**Контекст:** `MAX_COL_WIDTH = 105` при `MIN_COL_WIDTH = 100` давал диапазон в 5px — фактически колонка была всегда фиксированной.

**Решение:** Убран `Math.min(MAX_COL_WIDTH, w)` — колонка растягивается по самому длинному коду объекта. `CHAR_WIDTH` откалиброван под `fontSize: 12`.

**Риск:** При очень длинных кодах объектов (>15 символов) колонка займёт >120px, оставив меньше места для сетки месяцев. Принято как допустимое — читаемость важнее.
