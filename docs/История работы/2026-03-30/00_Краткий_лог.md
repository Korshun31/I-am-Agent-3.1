## Финальные фиксации (добавлено в конце дня)

### 1) Web bugfix: `Invalid Date` + регресс клика по свободной ячейке

- **Проблема:** календарь бронирований в Web показывал `Invalid Date`; после первого фикса перестал работать клик по свободной ячейке Gantt.
- **Причина:** невалидная X-координата клика и зависимость только от `locationX` в `react-native-web`.
- **Решение:** добавлены strict-guards для даты и безопасный резолвер X с fallback; клик по свободной ячейке восстановлен.
- **Проверка:** `Invalid Date` не воспроизводится, клик по ячейке снова открывает create, `+ Add booking` работает.
- **Итог:** обе web-проблемы закрыты.

### 2) Mobile bugfix: неверный месяц в create из `BookingCalendarScreen`

- **Проблема:** шаг календаря в `AddBookingModal` открывался с января 2025 (начало ленты), а не с текущего/выбранного месяца.
- **Причина:** расчёт месяца по локальному `locationX` без учёта горизонтального scroll offset.
- **Решение:** месяц теперь вычисляется по абсолютной координате (локальный X + scrollX) с guard на невалидный индекс.
- **Проверка:** после прокрутки таймлайна открывается корректный месяц; fallback на текущий месяц при невалидном key.
- **Итог:** mobile create-flow из календаря стабилизирован.

### 3) Role model checkpoint: финальная фиксация `admin/agent`

- **Проблема:** в документах оставались разночтения с legacy-терминами `owner`/`worker`.
- **Причина:** исторические записи и миграции использовали старые названия ролей.
- **Решение:** во всех ключевых docs зафиксировано: действующие роли только `admin`/`agent`; `owner` и `worker` помечены как legacy.
- **Проверка:** внесено SQL-подтверждение constraint’ов `company_members_role_check` и `company_members_status_check`.
- **Итог:** ролевая модель унифицирована в документации.

### 4) 3-я вкладка: Mobile Booking calendar UI parity

- **Проблема:** верхняя зона 3-й вкладки (mobile Booking calendar) визуально отличалась от паттерна `База`, из-за чего при переключении вкладок был заметный UI-разрыв.
- **Причина:** header/toolbar в `Booking calendar` были собраны в другом стиле (размеры/отступы/позиции элементов отличались от канонического mobile-паттерна).
- **Решение:** верхняя зона приведена к единому паттерну из `База`: 2 блока (`header` и `toolbar`), где `header = title center + bell right`, `toolbar = search + filter`; календарная сетка начинается ниже этих блоков; поиск фильтрует строки по `name/code`.
- **Проверка:** визуальный переход между `База` и `Booking calendar` стал консистентным; search оставляет релевантные дома/юниты; календарные действия (тап по ячейке, открытие/редактирование брони, детали) без регрессий.
- **Итог:** UI parity для 3-й вкладки зафиксирован, бизнес-логика не изменялась.
