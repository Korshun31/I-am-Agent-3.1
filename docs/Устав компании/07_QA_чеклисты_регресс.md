## Чеклист 1: Множественные отклонения

> Цель: убедиться, что история накапливается и UI обновляется без ручного refresh.

- [ ] Admin отклоняет объект #1 (reason="Причина 1") → история показывает 1 запись
- [ ] Agent отправляет повторно → Admin отклоняет #2 (reason="Причина 2") → история показывает 2 записи, #2 жирная
- [ ] Admin отклоняет #3 (reason="Причина 3") → 3 записи
- [ ] Admin отклоняет #4 (reason="Причина 3" — то же, что предыдущий) → 4 записи (одинаковая причина не мерджится)
- [ ] Admin отклоняет #5 → 5 записей, нумерация корректная (от 1 до 5 снизу вверх или от 5 до 1 сверху вниз — консистентно)
- [ ] Нет ошибок в консоли / alert'ов об ошибках схемы

## Чеклист 3: Approve после reject

- [ ] Admin отклоняет объект
- [ ] Admin нажимает Edit → сохраняет без правок → объект становится `approved`
- [ ] `rejection_reason` очищен в БД (проверить SQL: `SELECT rejection_reason FROM properties WHERE id=...`)
- [ ] `property_rejection_history` сохраняет старые записи (история не удалена)
- [ ] ⏳ чип исчезает в левой карточке
- [ ] Блок "История отклонений" скрыт (статус approved)
- [ ] Agent получает уведомление `property_approved`

## Чеклист 4: Уведомления

- [ ] Agent создаёт объект → Admin получает `property_submitted`
- [ ] Admin одобряет → Agent получает `property_approved`
- [ ] Admin отклоняет → Agent получает `property_rejected`
- [ ] Agent исправляет и отправляет → Admin получает `edit_submitted`
- [ ] Admin одобряет через edit (auto-approve) → Agent получает `property_approved`

## Чеклист 5: Проверка истории в БД

```sql
-- Проверить накопление истории по конкретному объекту:
SELECT id, reason, rejection_type, rejected_by, created_at
FROM property_rejection_history
WHERE property_id = '<property_uuid>'
ORDER BY created_at DESC;

-- Ожидаемый результат: N строк (по числу отклонений), каждая уникальная по created_at
```

- [ ] После reject #1: 1 строка
- [ ] После reject #2: 2 строки
- [ ] После auto-approve: строки сохраняются (не удалены)
- [ ] Поле `rejected_by` заполнено (не null, если auth.uid доступен)
