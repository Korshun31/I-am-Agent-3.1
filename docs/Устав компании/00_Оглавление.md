## Обязательное правило обновления

После каждого значимого изменения в проекте обновить **минимум два документа**:

1. `08_Журнал_решений_ADR.md` — зафиксировать решение, причину, альтернативы
2. Профильный документ по теме (SQL → `04`, RLS → `05`, бизнес-логика → `06`, инцидент → `10`)
